LANA RADGEN TRAINING COMMANDS

Environment
- Activate GPU environment:
  .\venv310\Scripts\Activate.ps1

- Or run directly without activating:
  .\venv310\Scripts\python scripts\train.py ...

- To enable W&B from .env, add:
  WANDB_API_KEY=your_wandb_api_key

- Optional Hugging Face token in .env:
  HF_TOKEN=your_huggingface_token

9. Method benchmark notes
These were measured on the current setup with:
- GPU: NVIDIA GeForce RTX 5070
- Model: DINOv3 ViT-S/16 + GPT-2
- Image size: 512
- Local batch size: 1

Results:
- lora_adamw: 0.1039s/step, 9.6234 img/s, loss 8.3437, fastest successful method
- lora_adamw8bit: 0.1215s/step, 8.2288 img/s, loss 9.8442
- full_adamw: 0.1222s/step, 8.1815 img/s, loss 9.3607
- full_adam: 0.3508s/step, 2.8509 img/s, loss 9.4696
- qlora_paged_adamw8bit: failed on the first CUDA training step with a device-side assert

Recommended training method:
- lora_adamw


10. Local batch size sweep notes
These were measured on the current setup with:
- Method: lora_adamw
- Image size: 512
- Device: CUDA

Results:
- batch 1: 0.2620s/step, 3.8174 img/s, peak allocated ~3.02 GB
- batch 2: 0.2406s/step, 8.3117 img/s, peak allocated ~5.61 GB
- batch 4: 0.7308s/step, 5.4733 img/s, peak allocated ~10.61 GB
- batch 6: 9.8556s/step, 0.6088 img/s, peak allocated ~15.63 GB
- batch 8: 20.7798s/step, 0.3850 img/s, peak allocated ~20.59 GB
- batch 10: CUDA OOM

What to remember:
- highest stable local batch found: 8
- fastest tested local batch: 2
- recommended practical local batch: 2
- use gradient accumulation if you want a larger effective batch without increasing local GPU memory too much

Example recommendation:
- local batch 2 + global batch 8


11. Global batch sweep notes
These were measured on the current setup with:
- Method: lora_adamw
- Local batch size: 2
- Image size: 512
- Device: CUDA

Results:
- global batch 2: accumulation 1, 0.3478s/optimizer step, 5.7500 img/s, peak allocated ~5.59 GB
- global batch 4: accumulation 2, 0.3830s/optimizer step, 10.4431 img/s, peak allocated ~5.60 GB
- global batch 8: accumulation 4, 0.7579s/optimizer step, 10.5558 img/s, peak allocated ~5.60 GB
- global batch 16: accumulation 8, 1.6283s/optimizer step, 9.8264 img/s, peak allocated ~5.60 GB
- global batch 32: accumulation 16, 3.2415s/optimizer step, 9.8719 img/s, peak allocated ~5.95 GB
- global batch 64: accumulation 32, 22.6055s/optimizer step, 2.8312 img/s, peak allocated ~6.16 GB

What to remember:
- highest tested global batch that worked: 64
- best throughput in this sweep: global batch 8
- global batch 4 was almost the same speed as 8
- global batch 64 is too slow to be practical on the current setup

Recommended practical setting:
- local batch 2
- global batch 8

Difference versus not using accumulation:
- global batch 2 means no accumulation
- global batch 8 keeps memory almost the same, but optimizer step time rises from 0.3478s to 0.7579s
- even with that, throughput improved from 5.7500 img/s to 10.5558 img/s in this sweep
